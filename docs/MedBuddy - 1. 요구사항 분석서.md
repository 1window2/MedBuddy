# (R1) 사용자 요구사항 정의서

## **1\. 개요 (Introduction)**

본 문서는 MedBuddy 시스템의 사용자 요구사항을 정의하는 것을 목적으로 한다.  
 MedBuddy는 처방전 및 약봉투 이미지를 분석하여 복용 정보를 사용자에게 직관적으로 제공하는 서비스이다.

현대의 약 처방 정보는 복잡한 형식과 어려운 용어로 인해 특히 노인 사용자 및 비전문가에게 이해하기 어려운 문제가 존재한다. 본 시스템은 이러한 문제를 해결하기 위해 이미지 기반 정보 추출과 사용자 친화적 인터페이스를 결합한 솔루션을 제공한다.

## **2\. 시스템 개요 (System Overview)**

MedBuddy는 사용자가 촬영한 처방전 이미지를 기반으로 다음과 같은 과정을 수행한다:

1. 이미지 입력 (카메라 촬영)  
2. OCR을 통한 텍스트 추출  
3. 텍스트 구조 분석 및 의미 파싱  
4. 복용 정보 구조화 (약 이름, 시간, 횟수 등)  
5. 사용자 친화적 형태로 출력 (텍스트 \+ 음성)

본 시스템은 모바일 환경을 기반으로 동작하며, 일부 민감 정보는 로컬에서 처리하여 개인정보 보호를 강화한다.

## **3\. 사용자 정의 (User Definition)**

| 사용자 유형 | 설명 |
| :---: | :---: |
| U1 | 노인 사용자 (처방전 이해가 어려운 사용자) |
| U2 | 보호자 (가족의 복약 관리를 돕는 사용자) |
| U3 | 간병인 (환자의 복약을 관리하는 사용자) |

## **4\. 기능 요구사항 (Functional Requirements)**

### **4.1 이미지 입력 및 처리**

\[FR-1\] 이미지 촬영 기능  
 사용자는 모바일 기기를 통해 처방전 또는 약봉투 이미지를 촬영할 수 있어야 한다.

\[FR-2\] 이미지 업로드 기능  
 사용자는 저장된 이미지를 업로드할 수 있어야 한다.

---

### **4.2 텍스트 인식 (OCR)**

\[FR-3\] 텍스트 추출 기능  
 시스템은 입력된 이미지로부터 텍스트를 추출해야 한다.

\[FR-4\] 다양한 형식 인식  
 시스템은 다양한 형태의 처방전 레이아웃을 인식할 수 있어야 한다.

---

### **4.3 처방 정보 파싱**

\[FR-5\] 약 정보 추출 기능  
 시스템은 텍스트에서 약 이름을 식별해야 한다.

\[FR-6\] 복용 정보 추출 기능  
 시스템은 복용 횟수, 복용 시간, 복용 기간을 추출해야 한다.

\[FR-7\] 구조화 기능  
 시스템은 추출된 정보를 JSON 형태의 구조화된 데이터로 변환해야 한다.

\[FR-8\] 레이아웃 기반 파싱 기능  
 시스템은 문서 구조를 고려하여 정보를 해석해야 한다.

---

### **4.4 사용자 인터페이스**

\[FR-9\] 정보 시각화 기능  
 시스템은 복용 정보를 직관적인 형태로 제공해야 한다.

\[FR-10\] 큰 글씨 UI 제공  
 시스템은 노인 사용자를 위해 큰 글씨 UI를 제공해야 한다.

---

### **4.5 음성 안내 기능**

\[FR-11\] 음성 출력 기능  
 시스템은 복용 정보를 음성으로 안내할 수 있어야 한다.

---

### **4.6 보안 및 데이터 처리**

\[FR-12\] 민감 정보 최소화 처리  
 시스템은 개인정보를 서버 전송 전에 최소화해야 한다.

\[FR-13\] 로컬 처리 기능  
 시스템은 가능한 경우 OCR 및 일부 처리를 로컬에서 수행해야 한다.

---

### **4.7 건강 관리 추천 기능**

\[FR-14\] 약물 기반 건강 추천 기능  
 시스템은 사용자가 복용 중인 약물 정보를 기반으로 운동 및 식습관 관련 추천을 제공해야 한다.

\[FR-15\] 약물 조합 고려 기능  
 시스템은 복수의 약물을 함께 복용하는 경우, 약물 간 상호작용을 고려한 맞춤형 추천을 제공해야 한다.

\[FR-16\] 사용자 맞춤 추천 기능  
 시스템은 사용자 상태(연령, 생활 패턴 등)를 반영하여 개인화된 건강 관리 정보를 제공할 수 있어야 한다.

## **5\. 비기능 요구사항 (Non-Functional Requirements)**

### **5.1 성능**

\[NFR-1\] 응답 시간  
 시스템은 OCR 및 결과 출력까지 3초 이내에 수행되어야 한다.

---

### **5.2 사용성 (Usability)**

\[NFR-2\] 직관적 UI  
 시스템은 IT 사용에 익숙하지 않은 사용자도 쉽게 사용할 수 있어야 한다.

\[NFR-3\] 접근성  
 글자 크기 및 UI 요소는 노인 사용자 기준으로 설계되어야 한다.

---

### **5.3 보안**

\[NFR-4\] 개인정보 보호  
 사용자의 민감 정보는 보호되어야 한다.

\[NFR-5\] 데이터 저장 제한  
 불필요한 개인정보는 저장하지 않아야 한다.

---

### **5.4 신뢰성**

\[NFR-6\] 인식 정확도  
 OCR 및 파싱 결과는 높은 정확도를 유지해야 한다.

## **6\. 제약사항 (Constraints)**

* 모바일 환경 기반 (Flutter)  
* OCR: Google ML Kit 사용  
* 서버: FastAPI  
* 데이터 관리: Firebase  
* 네트워크 연결이 불안정한 환경에서도 기본 기능 수행 필요 

## **7\. 향후 확장 고려사항**

* 약물 상호작용 경고 기능  
* LLM 기반 설명 기능  
* 사용자 맞춤 복약 알림 기능 

# (R2) 유스케이스 명세서

## **1\. 개요**

본 문서는 MedBuddy 시스템의 주요 유스케이스를 정의하고, 사용자가 시스템과 상호작용하는 흐름을 명확히 기술하는 것을 목적으로 한다.  
 각 유스케이스는 사용자 중심의 시나리오를 기반으로 시스템의 동작을 설명한다.

---

## **3\. 유스케이스 명세**

---

### **UC-1 처방전 촬영 및 분석**

Actor: 사용자

Precondition:

* 사용자는 MedBuddy 앱을 실행한 상태여야 한다

Main Flow:

1. 사용자가 카메라 기능을 실행한다  
2. 사용자가 처방전 또는 약봉투를 촬영한다  
3. 시스템이 이미지를 입력받는다  
4. 시스템이 OCR을 수행하여 텍스트를 추출한다  
5. 시스템이 텍스트를 분석하여 약 정보 및 복용 정보를 파싱한다  
6. 시스템이 구조화된 데이터를 생성한다

Postcondition:

* 처방 정보가 시스템 내부에 저장되고 다음 단계에서 활용 가능하다

---

### **UC-2 복용 정보 확인**

Actor: 사용자

Precondition:

* 처방전 분석이 완료된 상태여야 한다

Main Flow:

1. 사용자가 분석 결과 화면을 확인한다  
2. 시스템이 약 이름, 복용 시간, 횟수 등의 정보를 표시한다  
3. 사용자가 각 약에 대한 정보를 확인한다

Postcondition:

* 사용자는 자신의 복용 정보를 이해할 수 있다

---

### **UC-3 음성 안내 기능**

Actor: 사용자

Precondition:

* 복용 정보가 화면에 표시된 상태여야 한다

Main Flow:

1. 사용자가 음성 안내 기능을 선택한다  
2. 시스템이 복용 정보를 음성으로 변환한다  
3. 시스템이 사용자에게 음성으로 정보를 제공한다

Postcondition:

* 사용자는 음성을 통해 복용 정보를 이해할 수 있다

---

### **UC-4 건강 관리 추천 확인**

Actor: 사용자

Precondition:

* 사용자의 복용 정보가 시스템에 존재해야 한다

Main Flow:

1. 사용자가 건강 관리 추천 기능을 선택한다  
2. 시스템이 복용 중인 약물 정보를 분석한다  
3. 시스템이 약물 조합을 고려하여 운동 및 식습관 관련 추천을 생성한다  
4. 시스템이 사용자에게 추천 정보를 제공한다

Postcondition:

* 사용자는 자신의 건강 관리에 도움이 되는 정보를 제공받는다
